# 🎯 MOUP — 權證標的精選系統

> **Taiwan Warrant Intelligence System**
> 每日自動篩選台股中最具續漲潛力的 Top 10 標的，精準推薦可投資的認購權證

🌐 **線上網址：** [elvis2026-maker.github.io/MOUP](https://elvis2026-maker.github.io/MOUP/)

---

## 功能概覽

| 功能 | 說明 |
|------|------|
| 📋 **精選清單** | 每日盤後自動產出 Top 10 標的，顯示漲幅、均線、三大法人、續漲機率 |
| ⚡ **盤中即時確認** | 開盤後每 30 分鐘更新，確認精選股是否延續強勢，顯示開盤信號 |
| 🏷 **認購權證推薦** | 每支標的自動配對符合條件的認購權證（到期≥20天、槓桿5~12倍） |
| 📊 **評分系統** | 量價 40分 + 技術 30分 + 籌碼 30分，綜合評出 0~100 分 |

## 系統架構

```
GitHub Actions（自動排程）
  ├── fetch-data.yml  → 每日 16:00 / 18:00 / 18:30（台灣時間）
  │     └── fetch_twse.py  → 產出 data/stocks.json
  └── fetch-live.yml  → 盤中每 30 分鐘（09:05~13:25）+ 盤後 14:05
        └── fetch_live.py  → 產出 data/live.json

GitHub Pages（前端）
  └── index.html  → 讀取 stocks.json / live.json 顯示
```

## 評分邏輯

### 量價面（40分）
- 今日漲幅 ≥5% → +16分 / ≥3% → +12分 / ≥1% → +7分
- 收盤靠近當日最高點（收盤位置≥80%） → +14分
- 量能放大 1.5~4 倍 → +10分

### 技術面（30分）
- 均線多頭排列（MA5 > MA10 > MA20） → +10分
- 收盤站上月線（MA20） → +6分
- 突破近 10 日高點 → +10分

### 籌碼面（30分）
- 三大法人合計買超 >5000張 → +15分 / >1000張 → +10分
- 投信積極買超 >500張 → +5分
- 外資積極買超 >3000張 → +5分
- 融資追價過熱 → -5分

### 續漲機率對應分數

| 分數 | 機率等級 | 建議操作 |
|------|----------|----------|
| 85~100 | 🟢 高機率 | 積極操作認購權證 |
| 70~84  | 🔵 中高機率 | 小量試單，控制部位 |
| 55~69  | 🟡 中等機率 | 開盤觀察15分鐘後決定 |
| <55    | 🔴 偏低 | 不建議操作 |

## 掃描策略（V13 兩階段）

```
第一階段（快速粗篩）
  → 有 TOKEN：掃描 420 支
  → 無 TOKEN：掃描 220 支
  → 條件：近 2 天收漲、成交量達標、股價 10~5000 元
  → 通常淘汰 60~70%

第二階段（精細評分）
  → 對存活候選取近 35 天歷史資料
  → 計算完整均線、三大法人、融資評分
  → 選出 Top 10
```

## 部署步驟

### 1. Fork / Clone 此 Repo

### 2. 開啟 GitHub Pages
```
Settings → Pages → Source: Deploy from branch → main → / (root)
```

### 3. 設定 Actions 寫入權限
```
Settings → Actions → General → Workflow permissions → Read and write permissions ✅
```

### 4. 設定 FinMind Token（強烈建議）
至 [finmindtrade.com](https://finmindtrade.com) 免費申請帳號，取得 API Token 後：
```
Settings → Secrets and variables → Actions → New repository secret
  Name:  FINMIND_TOKEN
  Value: 你的 Token
```
> ⚠ 未設定 Token 僅能掃描 220 支股票（匿名限 300 req/hr），設定後可掃描 420 支（600 req/hr）

### 5. 手動觸發測試
```
Actions → 每日盤後抓資料 V13 → Run workflow
```

執行成功後，Actions log 最後一行應顯示：
```
✅ 完成！精選 N 支
```

## 認購權證篩選條件

展開個股卡片後，顯示符合以下條件的認購權證：

- 類型：**認購**（Call）
- 到期日：**距今 ≥ 20 天**
- 實質槓桿：**5~12 倍**（標示綠色；超出範圍標示橙色警示）
- 成交量：**≥ 50 張**（流動性門檻）
- 排序：Delta 在 0.45~0.65 的「價平」附近優先

## 常見問題

**Q：Actions 綠勾但網頁仍顯示空白？**
A：查看 Actions log，確認最後一行是否有 `✅ 完成`。若顯示 `FinMind 402`，表示 API 次數超限，建議設定 FINMIND_TOKEN。

**Q：盤中面板一直顯示「非交易時段」？**
A：正常現象。台股交易時間為週一至週五 09:00~13:30，非交易時段顯示昨日收盤資料。

**Q：精選結果只有幾支，不滿 10 支？**
A：當日符合條件的股票不足 10 支時，系統只顯示通過門檻的股票，不強制湊足 10 支。

**Q：資料超過 60 分鐘未更新？**
A：可能是 GitHub Actions 佇列延遲或 FinMind API 暫時異常，通常自動恢復。可至 Actions 頁面手動觸發。
